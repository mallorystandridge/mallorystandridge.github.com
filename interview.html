HTML Questions

  • What does a doctype do?

    • Specifies the rules for the markup language, so that the browsers render the content correctly.

  • What's the difference between standards mode and quirks mode?

    • There where legacy sites that existed before the W3C standards, quirks mode was used to keep old sites from breaking in newer browsers.

  • Are there any problems with serving pages as application/xhtml+xml?
  • How do you serve a page with content in multiple languages?

    • Your browser is configured to send accept-language headers, the web server can use those to determine your preferred language and send the appropriate document. Its it's called content negotiation

  • What kind of things must you be wary of when design or developing for multilingual sites?
    1. text direction
    2. bi-directional text
    3. different numbering systems
    4. sorting alphabetically
    5. how colors are perceived across different cultures

    You can see a lot more here https://en.wikipedia.org/

  • What are data-attributes good for?

    • They are pretty good for adding extra information to an element say like for js hooks that have no visual representation.

  • Consider HTML5 as an open web platform. What are the building blocks of HTML5?
    1. < header >
    2. < article >
    3. < aside >
    4. < nav >
    5. canvas
    6. new form elements like datepicker
    7. audio
    8. video
  • Describe the difference between a cookie, sessionStorage and localStorage.

    • cookie- works without js, the user can clear cookies or the cookie administrator chooses when the cookie is to expire.

    sessionStorage- only works with js, ends when ever the user close the tab.

    localStorage- only works with js, the user has to clear localStorage.

  • Describe the difference between < script >, < script async > and < script defer >.

    • < script >- blocks the html parser while the script downloads and executes

    < script async >- downloads without stoping the parser, but it does stop the parser when its finished downloading so it can run. Risking blocking rendering.

    < script defer >- doesn't run till all of the html is parsed and rendered.

  • Why is it generally a good idea to position CSS s between < head>< /head> and JS < script>s just before < /body >? Do you know any exceptions?

    • You would general want the layout with the css to render first and then js. If the js is in the header the user would have to wait for the js to fully load before rendering the layout.

  • What is progressive rendering?

    • Instead of giving the user a whole page at once, progressive rendering- renders sections of the content at a time.

  • Have you used different HTML templating languages before?

    • I have use markdown

CSS Questions

  • What is the difference between classes and ID's in CSS?

    • Elements can have multiple class but only one ID.

  • What's the difference between "resetting" and "normalizing" CSS? Which would you choose, and why?

    • Resetting would be like setting everything to zero. Normalizing would be like setting everything to a standardize values you are use to. I would use Normalize because keeps normal and useful defaults.

  • Describe Floats and how they work.

    • A float is a CSS positioning property. They allow for elements to be stacked side by side and not cause any reflow

  • Describe z-index and how stacking context is formed.

    • The z-index property specifies the stacking order of an element.

    A stacking context is formed...

    • the root element (HTML), positioned (absolutely or relatively) with a z-index value other than "auto",
    • a flex item with a z-index value other than "auto",that is the parent element display: flex|inline-flex,
    • elements with a transform value other than "none",
    • elements with a mix-blend-mode value other than "normal",
    • elements with a filter value other than "none",
    • elements with isolation set to "isolate",
  • Describe BFC(Block Formatting Context) and how it works.

    • It is the region in which the layout of block boxes occurs and in which floats interact with each other.
